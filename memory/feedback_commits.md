User stages and commits all changes themselves. Do not run git add, git commit, or git push commands.

**Why:** User wants full control over what goes into each commit and when it's pushed.

**How to apply:** When code changes are ready, stop and tell the user "ready to commit — changed files: X, Y, Z". Do not attempt to stage or commit anything.
